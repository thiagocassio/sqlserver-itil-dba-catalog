# 🔐 5.4.1 Falha de Certificados / Criptografia
**Categoria:** Operação e Segurança  
**CI afetado:** Certificado / Endpoint / Database  
**Escopo:** Segurança / Criptografia  
**Severidade sugerida:** P1 / P2  

---

## 🎯 Objetivo

Diagnosticar falhas relacionadas a criptografia em ambientes SQL Server, incluindo problemas com TDE (Transparent Data Encryption), certificados ou criptografia de conexões.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
SELECT
    name,
    is_encrypted
FROM sys.databases;
```
Objetivo:

-Identificar bancos com TDE habilitado

---

🧠 **Análise Detalhada**
- **Verificar Status da Criptografia**
```sql
SELECT
    db_name(database_id) AS database_name,
    encryption_state,
    percent_complete
FROM sys.dm_database_encryption_keys;
```
Objetivo:

- Verificar estado da criptografia

Estados comuns:

0 = No encryption
1 = Unencrypted
2 = Encryption in progress
3 = Encrypted

##

- **Verificar Certificados no Servidor**
```sql
SELECT
    name,
    subject,
    expiry_date
FROM sys.certificates;
```
Objetivo:

- Confirmar existência e validade do certificado

##

- **Verificar Endpoint de Criptografia**
```sql
SELECT
    name,
    protocol_desc,
    state_desc
FROM sys.endpoints;
```
Objetivo:

- Confirmar estado do endpoint de comunicação

---

🧩 **Possíveis Causas**

- Certificado expirado
- Certificado ausente após restore
- Chave mestra não restaurada
- Falha de configuração de endpoint
- Erro de configuração SSL

---

🛠️ **Ações de Mitigação**
- **Restaurar Certificado**
```sql
CREATE CERTIFICATE NomeCertificado
FROM FILE = 'caminho_certificado'
WITH PRIVATE KEY
(
FILE = 'caminho_chave',
DECRYPTION BY PASSWORD = 'senha'
);
```

##

- **Restaurar Chave Mestra**
```sql
RESTORE MASTER KEY
FROM FILE = 'caminho_masterkey'
DECRYPTION BY PASSWORD = 'senha';
```

##

- **Habilitar TDE**
```sql
ALTER DATABASE NomeDoBanco
SET ENCRYPTION ON;
```

---

🔁 **Monitoramento Pós-Mitigation**
```sql
SELECT
    db_name(database_id),
    encryption_state
FROM sys.dm_database_encryption_keys;
```

- Confirmar estado da criptografia
- Validar acesso ao banco
- Monitorar logs do SQL Server

---

📌 **Observação Operacional**

Sempre manter backup dos certificados e chaves utilizados pelo TDE.

Sem o certificado original:
- bancos criptografados não podem ser restaurados.
