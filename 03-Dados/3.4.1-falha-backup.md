# 💽 3.4.1 Falha de Backup
**Categoria:** Dados  
**CI afetado:** Plano de Backup / Job Backup  
**Escopo:** Proteção de Dados  
**Severidade sugerida:** P1 / P2  

---

## 🎯 Objetivo

Identificar falhas em rotinas de backup, analisando histórico de execução, mensagens de erro e integridade dos arquivos gerados.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
SELECT 
    database_name,
    backup_start_date,
    backup_finish_date,
    type,
    physical_device_name
FROM msdb.dbo.backupset bs
JOIN msdb.dbo.backupmediafamily bmf
ON bs.media_set_id = bmf.media_set_id
ORDER BY backup_start_date DESC;
```
---

🧠 **Análise Detalhada**
- **Verificar Histórico do Job**
```sql
SELECT 
    j.name,
    h.run_date,
    h.run_time,
    h.run_status,
    h.message
FROM msdb.dbo.sysjobhistory h
JOIN msdb.dbo.sysjobs j
ON h.job_id = j.job_id
WHERE j.name LIKE '%backup%'
ORDER BY h.run_date DESC, h.run_time DESC;
```
Objetivo:

- Identificar erro específico da execução

##

- **Verificar Espaço em Disco**

- Validar espaço disponível no destino
- Confirmar permissão da conta de serviço

##

- **Validar Integridade do Backup**
```sql
RESTORE VERIFYONLY
FROM DISK = 'caminho_do_backup.bak';
```
Objetivo:

- Confirmar integridade do arquivo gerado

---

🧩 **Possíveis Causas**

- Falta de espaço em disco
- Permissão insuficiente
- Caminho inválido
- Job desabilitado
- Corrupção no banco
- Timeout de I/O

---

🛠️ **Ações de Mitigação**

- Liberar espaço em disco
- Corrigir permissões
- Ajustar caminho de destino
- Executar backup manual para validação
- Revisar plano de manutenção

---

🔁 **Monitoramento Pós-Mitigation**
```sql
SELECT 
    database_name,
    MAX(backup_finish_date) AS LastBackup
FROM msdb.dbo.backupset
GROUP BY database_name;
```

- Validar execução bem-sucedida
- Implementar alerta de falha
- Monitorar rotina de backup diariamente

---

📌 **Observação Operacional**

- Backup não testado é backup inexistente.
- Além da execução, validar periodicamente a restauração em ambiente de teste.
