# 🔒 4.2.1 Bloqueios Prolongados
**Categoria:** Performance e Concorrência  
**CI afetado:** Sessão SQL / Database  
**Escopo:** Concorrência / Bloqueios  
**Severidade sugerida:** P1 / P2  

---

## 🎯 Objetivo

Identificar cadeias de bloqueio que impedem execução de consultas e impactam o desempenho ou disponibilidade do ambiente SQL Server.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
EXEC sp_whoisactive @find_block_leaders = 1;
```
Objetivo:

- Identificar sessões bloqueadas
- Detectar sessão líder do bloqueio (blocking session)

---

🧠 **Análise Detalhada**
- **Sessões Bloqueadas**
```sql
SELECT
    session_id,
    blocking_session_id,
    wait_type,
    wait_time,
    status,
    command
FROM sys.dm_exec_requests
WHERE blocking_session_id <> 0;
```
Objetivo:

- Identificar sessões bloqueadas e bloqueadoras

##

- **Cadeia de Bloqueio**
```sql
SELECT
    blocking_session_id AS blocking_session,
    session_id AS blocked_session,
    wait_type,
    wait_time
FROM sys.dm_exec_requests
WHERE blocking_session_id <> 0;
```
Objetivo:

- Identificar cadeia de bloqueios

##

- **Query Executada pela Sessão**
```sql
SELECT
    r.session_id,
    s.login_name,
    s.host_name,
    r.status,
    r.wait_type,
    r.blocking_session_id,
    t.text
FROM sys.dm_exec_requests r
JOIN sys.dm_exec_sessions s
ON r.session_id = s.session_id
CROSS APPLY sys.dm_exec_sql_text(r.sql_handle) t
WHERE r.blocking_session_id <> 0;
```
Objetivo:

- Identificar query responsável pelo bloqueio

---

🧩 **Possíveis Causas**

- Transações longas
- Falta de commit/rollback
- Operações de manutenção
- Índices inadequados
- Locks escalados

---

🛠️ **Ações de Mitigação**

- Identificar sessão bloqueadora
- Avaliar impacto da transação
- Solicitar finalização da transação na aplicação
- Encerrar sessão bloqueadora (último recurso)

##

- **Encerrar Sessão**
```sql
KILL <session_id>;
```
---

🔁 **Monitoramento Pós-Mitigation**
```sql
SELECT
    session_id,
    blocking_session_id
FROM sys.dm_exec_requests
WHERE blocking_session_id <> 0;
```
- Confirmar remoção da cadeia de bloqueio
- Monitorar novas ocorrências
- Avaliar necessidade de tuning ou revisão de transações

---

📌 **Observação Operacional**

Bloqueios são comportamento normal em sistemas transacionais.

O incidente ocorre quando:
- duração do bloqueio é excessiva
- número de sessões bloqueadas cresce rapidamente
- impacta processos críticos da aplicação
