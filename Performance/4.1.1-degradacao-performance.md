# ⚡ 4.1.1 Degradação Severa de Desempenho
**Categoria:** Performance e Concorrência  
**CI afetado:** Instância SQL / Database  
**Escopo:** Performance Geral  
**Severidade sugerida:** P1 / P2  

---

## 🎯 Objetivo

Identificar consultas, waits e recursos responsáveis por degradação severa de desempenho no SQL Server.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
EXEC sp_whoisactive;
```
Objetivo:

- Identificar sessões ativas
- Detectar bloqueios e queries pesadas

---

🧠 **Análise Detalhada**
- **Queries com maior consumo de CPU**
```sql
SELECT TOP 10
    total_worker_time / execution_count AS avg_cpu,
    execution_count,
    qs.total_worker_time,
    qs.total_elapsed_time,
    SUBSTRING(st.text,
    (qs.statement_start_offset/2)+1,
    ((CASE qs.statement_end_offset
        WHEN -1 THEN DATALENGTH(st.text)
        ELSE qs.statement_end_offset
    END - qs.statement_start_offset)/2)+1) AS query_text
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) st
ORDER BY avg_cpu DESC;
```
Objetivo:

- Identificar consultas que mais consomem CPU

##

- **Análise de Wait Statistics**
```sql
SELECT TOP 20
    wait_type,
    wait_time_ms,
    waiting_tasks_count
FROM sys.dm_os_wait_stats
ORDER BY wait_time_ms DESC;
```
Objetivo:

- Identificar tipo de contenção dominante

##

- **Sessões Ativas**
```sql
SELECT
    session_id,
    status,
    cpu_time,
    logical_reads,
    blocking_session_id,
    wait_type
FROM sys.dm_exec_requests
WHERE session_id > 50;
```
Objetivo:

- Identificar sessões consumindo recursos

---

🧩 **Possíveis Causas**

- Query mal otimizada
- Falta de índices
- Estatísticas desatualizadas
- Contenção de recursos
- Plano de execução ruim
- Mudança recente de deploy

---

🛠️ **Ações de Mitigação**

- Identificar query dominante
- Avaliar plano de execução
- Atualizar estatísticas
- Criar índice adequado
- Avaliar regressão de plano

---

🔁 **Monitoramento Pós-Mitigation**
```sql
SELECT
    wait_type,
    wait_time_ms
FROM sys.dm_os_wait_stats
ORDER BY wait_time_ms DESC;
```

- Monitorar waits predominantes
- Validar redução de consumo de CPU
- Confirmar normalização da performance

---

📌 **Observação Operacional**

Nem sempre o problema é o SQL Server.
Performance também pode ser impactada por:
- Storage
- CPU do host
- Rede
- Aplicação
