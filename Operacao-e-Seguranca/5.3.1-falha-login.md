# 🔑 5.3.1 Falha de Conexão / Autenticação
**Categoria:** Operação e Segurança  
**CI afetado:** Login SQL / Active Directory  
**Escopo:** Autenticação / Conectividade  
**Severidade sugerida:** P2 / P3  

---

## 🎯 Objetivo

Diagnosticar falhas de autenticação em conexões ao SQL Server, identificando erros relacionados a logins SQL ou autenticação via Active Directory.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

Erro comum retornado pelo SQL Server:

Login failed for user
Error 18456

---

## 🧠 Análise Detalhada

###  Verificar Logins Existentes

```sql
SELECT
    name,
    type_desc,
    is_disabled
FROM sys.server_principals
WHERE type IN ('S','U','G');
``
Objetivo:

- Confirmar existência do login
- Verificar se o login está desabilitado

##

- **Verificar Banco Padrão**
``sql
SELECT
    name,
    default_database_name
FROM sys.server_principals
WHERE name = 'NomeDoLogin';
``
Objetivo:

- Verificar se o banco padrão existe ou está acessível

##

- **Verificar Error Log**
``sql
EXEC xp_readerrorlog;
``
Objetivo:

- Identificar código de estado do erro 18456

---

🧩 **Possíveis Causas**

- Login inexistente
- Login desabilitado
- Senha incorreta
- Banco padrão inacessível
- Problemas de autenticação AD
- Modo de autenticação incorreto (SQL / Windows)

---

🛠️ **Ações de Mitigação**
- **Habilitar Login**
``sql
ALTER LOGIN NomeDoLogin ENABLE;
``

##

- **Alterar Senha**
``sql
ALTER LOGIN NomeDoLogin
WITH PASSWORD = 'NovaSenha';
``

##

- **Alterar Banco Padrão**
``sql
ALTER LOGIN NomeDoLogin
WITH DEFAULT_DATABASE = master;
``

---

🔁 **Monitoramento Pós-Mitigation**

- Validar nova tentativa de conexão
- Monitorar Error Log
- Confirmar acesso ao banco necessário

---

📌 **Observação Operacional**

Falhas de login podem ocorrer devido a mudanças recentes como:

- alteração de senha
- migração de banco
- mudança de autenticação
- alteração de permissões
