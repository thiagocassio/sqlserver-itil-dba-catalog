# 🔐 5.3.2 Falha de Permissões / Acesso a Objetos
**Categoria:** Operação e Segurança  
**CI afetado:** Database / Schema / Objeto  
**Escopo:** Autorização / Controle de Acesso  
**Severidade sugerida:** P2 / P3  

---

## 🎯 Objetivo

Diagnosticar falhas de acesso a objetos do banco de dados, identificando permissões ausentes ou configuração incorreta de usuários e roles.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

Erro comum retornado pelo SQL Server:

- The SELECT permission was denied on the object

---

## 🧠 Análise Detalhada

### - Verificar Usuário no Banco

```sql
SELECT
    name,
    type_desc
FROM sys.database_principals
WHERE name = 'NomeDoUsuario';
```
Objetivo:

- Confirmar existência do usuário no banco

##

- **Verificar Permissões do Usuário**
```sql
SELECT
    dp.name,
    dp.type_desc,
    pe.permission_name,
    pe.state_desc
FROM sys.database_permissions pe
JOIN sys.database_principals dp
ON pe.grantee_principal_id = dp.principal_id
WHERE dp.name = 'NomeDoUsuario';
```
Objetivo:

- Identificar permissões concedidas ou negadas

##

- **Verificar Roles do Usuário**
```sql
SELECT
    dp.name AS RoleName,
    mp.name AS MemberName
FROM sys.database_role_members drm
JOIN sys.database_principals dp
ON drm.role_principal_id = dp.principal_id
JOIN sys.database_principals mp
ON drm.member_principal_id = mp.principal_id
WHERE mp.name = 'NomeDoUsuario';
```
Objetivo:

- Identificar roles atribuídas

---

🧩 **Possíveis Causas**

- Usuário não mapeado ao login
- Permissão não concedida
- Permissão negada explicitamente
- Mudança recente de role
- Alteração de schema ou objeto

---

🛠️ **Ações de Mitigação**
- **Conceder Permissão**
```sql
GRANT SELECT
ON NomeDoSchema.NomeDaTabela
TO NomeDoUsuario;
```

##

- **Adicionar Usuário a Role**
```sql
ALTER ROLE db_datareader
ADD MEMBER NomeDoUsuario;
```

##

- **Mapear Login a Usuário**
```sql
CREATE USER NomeDoUsuario
FOR LOGIN NomeDoLogin;
```

---

🔁 **Monitoramento Pós-Mitigation**

- Validar execução da consulta pelo usuário
- Confirmar acesso ao objeto necessário
- Revisar políticas de segurança

---

📌 **Observação Operacional**

Permissões devem seguir o princípio de menor privilégio.

Evitar concessão excessiva como:
- db_owner
- sysadmin
