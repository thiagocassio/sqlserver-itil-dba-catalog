# 🔄 5.2.1 Falha em ETL / Carga Crítica
**Categoria:** Operação e Segurança  
**CI afetado:** Pacote ETL / Job SQL Agent  
**Escopo:** Processamento de Dados / Integração  
**Severidade sugerida:** P1 / P2  

---

## 🎯 Objetivo

Identificar falhas em processos de ETL ou cargas críticas de dados executadas por SQL Server Agent, SSIS ou scripts de integração.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
SELECT
    j.name,
    h.run_date,
    h.run_time,
    h.run_status,
    h.message
FROM msdb.dbo.sysjobhistory h
JOIN msdb.dbo.sysjobs j
ON h.job_id = j.job_id
ORDER BY h.run_date DESC, h.run_time DESC;
```
Objetivo:

- Identificar job responsável pela carga
- Verificar erro ocorrido durante execução

---

🧠 **Análise Detalhada**
- **Identificar Etapa do Job**
```sql
EXEC msdb.dbo.sp_help_jobhistory
    @job_name = 'NomeDoJob';
```
Objetivo:

- Identificar etapa exata da falha

##

- **Verificar Logs do Processo ETL**

- Dependendo da ferramenta utilizada:
- Logs do SSIS
- Logs da aplicação
- Tabelas de auditoria de carga

Objetivo:

- Identificar erro funcional ou de dados

##

- **Verificar Sessões em Execução**
```sql
SELECT
    session_id,
    status,
    cpu_time,
    logical_reads,
    blocking_session_id,
    wait_type
FROM sys.dm_exec_requests
WHERE session_id > 50;
```
Objetivo:

- Identificar sessões relacionadas ao processo ETL

---

🧩 **Possíveis Causas**

- Falha de conexão com origem/destino
- Problemas de formato de dados
- Timeout durante carga
- Bloqueios no banco de dados
- Falha de permissão

---

🛠️ **Ações de Mitigação**

- Revisar erro específico no log do processo
- Executar etapa manualmente para diagnóstico
- Validar conectividade com origem e destino
- Corrigir inconsistências de dados

---

🔁 **Monitoramento Pós-Mitigation**
```sql
SELECT
    name,
    enabled
FROM msdb.dbo.sysjobs;
```

- Validar execução do job após correção
- Monitorar novas cargas
- Revisar alertas de falha de ETL

---

📌 **Observação Operacional**

Processos de ETL frequentemente suportam:

- relatórios
- integrações entre sistemas
- cargas de data warehouse

Falhas podem impactar diretamente processos de negócio.
