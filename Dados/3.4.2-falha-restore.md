# 🔄 3.4.2 Falha de Restore
**Categoria:** Dados  
**CI afetado:** Database / Arquivo de Backup  
**Escopo:** Recuperação / Continuidade  
**Severidade sugerida:** P1  

---

## 🎯 Objetivo

Identificar falhas durante o processo de restauração de banco de dados, validando cadeia de backup, compatibilidade e integridade dos arquivos.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
RESTORE VERIFYONLY
FROM DISK = 'caminho_do_backup.bak';
```
---

🧠 **Análise Detalhada**
-  **Validar Header do Backup**
```sql
RESTORE HEADERONLY
FROM DISK = 'caminho_do_backup.bak';
```
Objetivo:

- Confirmar tipo de backup (FULL, DIFF, LOG)
- Validar compatibilidade de versão

##

- **Validar Logical Files**
```sql
RESTORE FILELISTONLY
FROM DISK = 'caminho_do_backup.bak';
```
Objetivo:

- Identificar nomes lógicos dos arquivos
- Preparar comando RESTORE com MOVE

##

- **Cadeia de Backup (FULL + DIFF + LOG)**
```sql
SELECT 
    database_name,
    type,
    backup_start_date,
    backup_finish_date
FROM msdb.dbo.backupset
WHERE database_name = 'NomeDoBanco'
ORDER BY backup_start_date;
```
Objetivo:

- Confirmar sequência correta de restauração

---

🧩 **Possíveis Causas**

- Backup corrompido
- Ordem incorreta de restauração
- Falta de backup diferencial ou log
- Banco em uso durante restore
- Espaço insuficiente em disco
- Versão incompatível

---

🛠️ **Ações de Mitigação**

1️⃣ **Restaurar em Ordem Correta**
```sql
RESTORE DATABASE NomeDoBanco
FROM DISK = 'backup_full.bak'
WITH NORECOVERY;
```
```sql
RESTORE LOG NomeDoBanco
FROM DISK = 'backup_log.trn'
WITH RECOVERY;
```
##

2️⃣ **Restaurar com MOVE (Quando necessário)**
```sql
RESTORE DATABASE NomeDoBanco
FROM DISK = 'backup_full.bak'
WITH MOVE 'LogicalDataName' TO 'caminho.mdf',
     MOVE 'LogicalLogName'  TO 'caminho.ldf',
     REPLACE,
     RECOVERY;
```
##

3️⃣ **Forçar Single User (Quando bloqueado)**
```sql
ALTER DATABASE NomeDoBanco
SET SINGLE_USER WITH ROLLBACK IMMEDIATE;
```
---

🔁 **Monitoramento Pós-Mitigation**
```sql
SELECT 
    name,
    state_desc
FROM sys.databases
WHERE name = 'NomeDoBanco';
```

- Confirmar banco ONLINE
- Validar integridade com DBCC CHECKDB
- Testar aplicação

---

📌 **Observação Operacional**

- Restore é o único teste real de um backup.
- Sem testes periódicos de restauração, a estratégia de backup não está validada.
