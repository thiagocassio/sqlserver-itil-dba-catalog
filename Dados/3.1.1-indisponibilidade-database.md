# 🚫 3.1.1 Indisponibilidade do Banco de Dados
**Categoria:** Dados  
**CI afetado:** Database específico  
**Escopo:** Disponibilidade / Estado do Banco  
**Severidade sugerida:** P1  

---

## 🎯 Objetivo

Identificar bancos de dados inacessíveis ou em estados anormais, como OFFLINE, SUSPECT, RECOVERY_PENDING ou EMERGENCY.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
SELECT 
    name,
    state_desc,
    recovery_model_desc
FROM sys.databases;
```
---

🧠 **Análise por Estado do Banco**
- **Bancos Não Online**
```sql
SELECT 
    name,
    state_desc
FROM sys.databases
WHERE state_desc <> 'ONLINE';
```
Objetivo:

- Identificar bancos fora do estado normal

##

- **Verificar Motivo no Error Log**
```sql
EXEC xp_readerrorlog;
```
Objetivo:

- Localizar erro raiz da indisponibilidade

##

- **Verificar Arquivos Físicos**
```sql
SELECT 
    name,
    physical_name,
    state_desc
FROM sys.database_files;
```
Objetivo:

- Confirmar existência e acessibilidade dos arquivos MDF/LDF

---

🧩 **Possíveis Causas**

- Falha de storage
- Corrupção de arquivos
- Falha durante recovery
- Log inconsistente
- Permissão alterada no sistema de arquivos

---

🛠️ **Ações de Mitigação**

- Verificar integridade do storage
- Colocar banco em EMERGENCY (quando necessário)
- Executar DBCC CHECKDB
- Restaurar a partir de backup válido
- Validar espaço em disco

---

🔁 **Monitoramento Pós-Mitigation**
```sql
SELECT 
    name,
    state_desc
FROM sys.databases;
```
