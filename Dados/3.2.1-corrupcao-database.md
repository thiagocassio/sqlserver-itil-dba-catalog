# 🛡️ 3.2.1 Corrupção de Banco de Dados
**Categoria:** Dados  
**CI afetado:** Database  
**Escopo:** Integridade Estrutural  
**Severidade sugerida:** P1  

---

## 🎯 Objetivo

Identificar e diagnosticar corrupção estrutural em banco de dados utilizando ferramentas nativas do SQL Server, avaliando impacto e estratégia de recuperação.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
DBCC CHECKDB ('NomeDoBanco') WITH NO_INFOMSGS, ALL_ERRORMSGS;
```
---

🧠 **Análise Detalhada**
- **Verificação de Integridade**
```sql
DBCC CHECKDB ('NomeDoBanco') WITH NO_INFOMSGS, ALL_ERRORMSGS;
```
Objetivo:

- Detectar corrupção em páginas de dados ou índices
- Identificar erros 823, 824 ou 825

##

- **Verificar Erros no Error Log**
```sql
EXEC xp_readerrorlog;
```
Objetivo:

- Localizar mensagens relacionadas a I/O ou corrupção

##

- **Identificar Objetos Afetados**
```sql
DBCC CHECKDB ('NomeDoBanco') WITH TABLERESULTS;
```
Objetivo:

- Identificar tabela, índice ou alocação afetada

---

🧩 **Possíveis Causas**

- Falha de storage
- Problemas físicos em disco
- Falha de energia
- Erros de firmware
- Snapshot inconsistente
- Corrupção em backup

---

🛠️ **Ações de Mitigação**
1️⃣ **Avaliar Severidade**

- Confirmar se há backup íntegro recente
- Avaliar impacto no negócio

##

2️⃣ **Restaurar Backup (Recomendado)**
```sql
RESTORE DATABASE NomeDoBanco
FROM DISK = 'caminho_do_backup'
WITH REPLACE, RECOVERY;
```
##

3️⃣ **Reparação Emergencial (Último Recurso)**
```sql
ALTER DATABASE NomeDoBanco SET SINGLE_USER WITH ROLLBACK IMMEDIATE;
```
```sql
DBCC CHECKDB ('NomeDoBanco', REPAIR_ALLOW_DATA_LOSS);
```
```sql
ALTER DATABASE NomeDoBanco SET MULTI_USER;
```

⚠️ **Utilizar apenas quando não houver backup válido.**
---

🔁 **Monitoramento Pós-Mitigation**
```sql
DBCC CHECKDB ('NomeDoBanco');
```
- Validar execução sem erros
- Monitorar Error Log
- Implementar rotina periódica de CHECKDB

---

📌 **Observação Operacional**

- Corrupção de banco de dados é incidente crítico de infraestrutura ou storage até prova em contrário.

- Sempre validar camada física antes de executar reparo lógico.
