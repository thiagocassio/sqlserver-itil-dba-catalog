# ⚠️ 4.2.2 Deadlocks
**Categoria:** Performance e Concorrência  
**CI afetado:** Database / Stored Procedure  
**Escopo:** Concorrência / Transações  
**Severidade sugerida:** P1 / P2  

---

## 🎯 Objetivo

Identificar conflitos entre transações que resultam em deadlock, analisando sessões envolvidas, recursos bloqueados e ordem de execução.

---

## 🔎 Diagnóstico Inicial (Quick Triage)

```sql
SELECT *
FROM sys.dm_os_waiting_tasks
WHERE wait_type LIKE 'LCK%';
```
Objetivo:

- Detectar sessões aguardando locks

---

🧠 **Análise Detalhada**
- **Deadlocks Registrados no Error Log**
```sql
EXEC xp_readerrorlog;
```
Objetivo:

- Identificar ocorrência recente de deadlock

##

- **Captura de Deadlock com Extended Events**
```sql
CREATE EVENT SESSION DeadlockMonitor
ON SERVER
ADD EVENT sqlserver.xml_deadlock_report
ADD TARGET package0.event_file
(
    SET filename = 'deadlock_monitor'
);
ALTER EVENT SESSION DeadlockMonitor
ON SERVER
STATE = START;
```
Objetivo:

- Capturar deadlock graph automaticamente

##

- **Consulta aos Eventos Capturados**
```sql
SELECT
    XEvent.query('(event/data/value/deadlock)[1]') AS DeadlockGraph
FROM
(
    SELECT CAST(event_data AS XML) AS XEvent
    FROM sys.fn_xe_file_target_read_file('deadlock_monitor*.xel', NULL, NULL, NULL)
) AS Deadlocks;
```
Objetivo:

- Visualizar grafo do deadlock

---

🧩 **Possíveis Causas**

- Acesso concorrente aos mesmos recursos
- Ordem diferente de acesso a tabelas
- Transações longas
- Falta de índices adequados

---

🛠️ **Ações de Mitigação**

- Revisar ordem de acesso às tabelas
- Reduzir duração das transações
- Criar índices adequados
- Ajustar nível de isolamento
- Revisar lógica da aplicação

---

🔁 **Monitoramento Pós-Mitigation**
```sql
SELECT *
FROM sys.dm_os_wait_stats
WHERE wait_type LIKE 'LCK%';
```

- Monitorar waits relacionados a lock
- Validar redução de deadlocks
- Revisar métricas de concorrência

---

📌 **Observação Operacional**

Deadlocks não são necessariamente erros de infraestrutura.

Eles geralmente indicam:
- concorrência mal planejada
- design de transação inadequado
- ausência de índices adequados
